# `PLAN.md` Scaffold

Use `PLAN.md` for the stable end-state target, not routine progress.

## Recommended Structure

### 1. Project Identity

- project name
- positioning or short statement of what it is
- core objective

### 2. Target Outcome

Describe what the finished system should do and what the intended shape of the project is.

### 3. Scope Boundaries

Separate:

- what the project must do
- what is explicitly deferred

### 4. Architecture Direction

Capture major implementation choices that shape the final system:

- canonical artifacts or data model
- platform targets
- major dependency direction
- integration boundaries
- important constraints

### 5. Roadmap or Milestones

Describe the milestone-level journey from current state to target state.

### 6. Open Questions

Keep strategic or cross-cutting unresolved questions visible here until they are answered or become backlog decision gates.

## Quality Bar

If someone reads only `PLAN.md`, they should understand where the project is headed and what kind of system they are building.
