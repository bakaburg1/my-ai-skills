# Bootstrap Discovery Checklist

Use this checklist before writing the canonical coordination files. Extract what the user already gave you, then ask only for the missing information that would materially improve the scaffold.

## Minimum Backbone

- project name
- short project description
- purpose or problem being solved
- intended audience or users
- target deliverable or product shape
- major scope boundaries
- major technical design or constraints
- initial milestones or phases
- known open questions
- what the user wants to defer

## How to Use It

1. Fill from the prompt and repo first.
2. Ask only for missing high-impact facts.
3. If the user defers a choice, do not guess it.
4. Route deferred choices into the right artifact:
   - `PLAN.md` for strategic or cross-cutting questions
   - `BACKLOG.md` for future decision gates
   - memory for durable unresolved context

## Backbone Quality Test

Do not start writing until you can explain, in plain language:

- what this project is
- what success looks like
- what the first implementation milestones should roughly cover
- which major questions are still open

If you cannot do that, the scaffold is not ready.
